# Contract: Customer Value Mapping Module (客户价值映射模块)

**Module**: Module 6 - Customer Value Mapping
**Status**: Contract Defined

## Module Purpose
针对两类核心客户,定制化阐述价值主张、决策因素和价值实现路径。

## Output Specifications

### 1. Customer Type Value Maps (2类客户)

#### 企业IT部门

**Decision Factors** (排序,至少4个):
1. 成本节约(ROI) - How: 开发成本降低50%,维护成本降低40%
2. 风险可控(安全/合规) - How: 企业级安全架构,数据本地化部署
3. 技术前瞻性(避免技术债) - How: AI原生架构,持续进化无需重构
4. 厂商可靠性(长期支持) - How: 独立新品牌,长期技术路线图

**Value Realization Path** (至少4个Milestone):
- Month 1: POC验证(简单应用如会议室预约)
- Month 3: 首个生产应用(内部CRM/OA)
- Month 6-12: 规模化应用(5-10个内部系统)
- Year 2+: 企业应用平台(数字化转型支撑)

#### 系统集成商

**Decision Factors** (排序,至少4个):
1. 利润率提升(直接影响盈利) - How: 人力成本降低30-50%,利润率从20%提升到40-50%
2. 交付速度(竞争力) - How: 项目周期缩短60-80%
3. 人员依赖度降低(风险) - How: 标准化交付,人员流动影响降低80%
4. 可复制性(规模化) - How: 同样团队承接项目数增加3-5倍

**Value Realization Path** (至少4个Milestone):
- Month 1: 内部试点(验证效率提升)
- Month 2-3: 客户试点(1-2个中小型项目)
- Month 6: 标准化交付(建立AI平台交付流程)
- Year 1+: 规模化复制(50%以上新项目使用AI平台)

### 2. Unsuitable Scenarios (至少4个)

必须诚实说明不适合场景:
1. 超高性能要求系统(高频交易/实时控制)
2. 极致定制化复杂业务逻辑(AI理解能力限制)
3. 强监管行业核心系统初期阶段(监管框架待完善)
4. 超大规模企业级系统MVP阶段(聚焦中小型应用)

每个场景必须包含:
- 场景描述
- 不适合原因
- 建议方案

## Acceptance Criteria

- [ ] 2类客户完整映射
- [ ] Decision Factors有明确排序和"How we address"
- [ ] Value Realization Path按时间顺序,至少4个Milestone
- [ ] Unsuitable Scenarios诚实说明,不回避局限性
- [ ] 所有内容与ValueProposition模块一致
