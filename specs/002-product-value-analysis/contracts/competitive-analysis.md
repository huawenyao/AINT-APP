# Contract: Competitive Analysis Module (竞品对比分析模块)

**Module**: Module 3 - Competitive Analysis
**Status**: Contract Defined

## Module Purpose
系统性对比5类竞争对手,凸显AI原生平台的差异化优势,支撑"为什么选择我们"的论述。

## Output Specifications

### 1. Competitive Matrix (竞品对比矩阵)

**5个竞争对手类别**(至少覆盖):
1. Traditional Low-Code (OutSystems/Mendix/Power Platform)
2. Chinese Low-Code (宜搭/氚云/简道云)
3. AI Coding Tools (GitHub Copilot/Cursor)
4. Custom Development (外包/自研)
5. Reference (NocoBase - 技术参照)

**7个对比维度**:
- 开发效率 (Development Efficiency)
- 学习成本 (Learning Cost)
- 灵活性 (Flexibility)
- 可维护性 (Maintainability)
- 适用场景 (Use Cases)
- 技术门槛 (Technical Barrier)
- AI能力深度 (AI Capability Depth)

**评分标准**: excellent / good / fair / poor

### 2. Why Choose Us (为什么选择我们)

每个竞争对手类别必须包含:
- 至少3个关键差异点
- 至少2个实际场景示例
- 明确的对比结果差异

### 3. Competitor Profiles (每类至少1个代表)

必须包含字段:
- 核心交互模式
- 技术架构特点
- 关键局限性(至少2个)
- 目标客户群体
- 大致定价范围(如可获取)

## Acceptance Criteria

- [ ] 5个竞争对手类别完整覆盖
- [ ] 7个对比维度完整评分
- [ ] 每个评分有rationale和evidence
- [ ] "Why Choose Us"论述清晰有力
- [ ] 所有数据可追溯到research.md
